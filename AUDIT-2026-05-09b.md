### ❌ FIXED THIS AUDIT

**Medium: SameSite=None → SameSite=Lax**

The JWT cookie was set with `SameSite=None` after the migration. While functional, this is unnecessarily permissive. `citeback.com` and `ai.citeback.com` share the same eTLD+1 (`citeback.com`), making them **same-site** by the browser's definition. SameSite=None means the cookie is sent with *all* cross-site requests — including form submissions from attacker-controlled sites, enabling logout CSRF or future sensitive-action CSRF as features are added.

With SameSite=Lax:
- All `credentials: 'include'` fetches from `citeback.com → ai.citeback.com` still work (same-site)
- Cookie is NOT sent with cross-site POST/PATCH requests (CSRF prevention)
- No user impact or functionality change

**Fix applied:**
- `server/server.js`: `SameSite=None` → `SameSite=Lax` in `COOKIE_OPTS` and `COOKIE_CLEAR`
- Live Hetzner server: same change applied via `sed`, service restarted
- Commit: `aaed35f`
