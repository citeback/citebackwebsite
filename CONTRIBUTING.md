# Contributing to Citeback

Thanks for your interest. Citeback is a surveillance resistance platform — here's how you can help.

## Ways to Contribute

### 1. Run a Campaign (Highest Impact)
Operators claim and run campaigns — FOIA requests, billboard campaigns, legal defense funds. If you have a target and the capacity to execute, [apply to become an operator](https://citeback.com/run-a-campaign).

### 2. Contribute Expertise
We need attorneys, FOIA specialists, journalists, and privacy researchers for the [Expert Directory](https://citeback.com/expert-directory). These are not code roles — they support active campaigns.

### 3. Submit Camera Sightings
If you spot an ALPR camera or surveillance device, submit it via the [camera map](https://citeback.com/map). C2PA-verified photos (Proofmode, Samsung Galaxy S24+, Pixel 10) earn reputation points.

### 4. Fund a Campaign
No account needed. Contributions go directly to operator wallets via Monero (XMR) or Zano (ZANO).

---

## Code Contributions

**The codebase is not open for outside pull requests.**

This is a deliberate security decision. The platform handles privacy-sensitive data and crypto wallet references. Allowing outside PRs would create a vector for supply chain attacks against the cryptographic verification layer (C2PA, wallet address display).

If you find a security vulnerability, see [SECURITY.md](./SECURITY.md).

If you find a bug or have a feature idea, open an issue — we'll handle it internally.

---

## Governance Contributions

The governance model ([GOVERNANCE.md](./GOVERNANCE.md)) and mission constitution are open documents. If you have substantive feedback on governance rules, disbursement logic, or operator accountability, open an issue with the `governance` label.

Rule changes require community votes after launch (see GOVERNANCE.md §12).
